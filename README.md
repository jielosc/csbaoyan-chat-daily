# CS Baoyan Chat Daily 📰

这是一个面向 CS 保研群（[绿群](https://github.com/CS-BAOYAN)）的每日 AI 信息总结项目。

做这个项目，不是为了制造更多信息，而是想用 AI 把群里那些分散的经验、通知、提醒和讨论，整理成每天都能看、也方便回头翻的日报，打破一点信息差。

在线查看 Pages：

- https://jielosc.github.io/csbaoyan-chat-daily

## 免责声明 ⚠️

本项目产出的日报，本质上是对群聊信息的整理、归纳和再表达，主要用于信息汇总与交流参考，不应视为官方结论或正式通知。

由于原始信息来自群聊讨论，且整理过程中包含 AI 总结与重写环节，因此内容可能存在时效性变化、表达偏差、信息遗漏，或总结不完全准确的情况，阅读时还请注意甄别。

如涉及报名要求、夏令营/预推免安排、材料提交、时间节点等关键信息，请始终以学校、院系、导师或官方平台发布的内容为准。

## 项目主要流程 🔄

整个流程大概是这样：

1. 读取已经导出的 QQ 群聊天 JSON
2. 对聊天内容做清洗和匿名化处理，尽量保留信息、去掉不必要的身份暴露
3. 按主题和上下文分块提取有效内容，过滤灌水与重复信息
4. 用 AI 将分块结果进一步整理，生成结构化的每日日报
5. 把最终日报同步到静态页面目录，方便归档和公开浏览

运行过程中，产物大概分两类：

- `internal/`：本地生成的中间产物，比如脱敏后的聊天记录、提取草稿等，默认不进入 GitHub 仓库
- `pages/data/reports/`：最终公开的日报内容，便于归档和浏览

## 为什么做这个项目 🤔

保研相关的信息天然就很分散，常见情况大概是：

- 重要通知、经验分享和零散讨论常常混在大量聊天消息里，后加入的人很难补课，也不容易系统整理
- 绿群本身有人数上限，会定期清理长期不发言的人，不是所有人都能一直留在群里
- 群消息量非常大，普通人很难长期、高频地完整跟进

所以这个项目想做的事其实很直接：把群聊里那些高价值、时效性强的信息尽量沉淀下来，降低后来者获取信息的门槛，也少一点因为信息不对称带来的额外成本。
